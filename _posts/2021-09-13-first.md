## 쿠키
웹사이트 접속 시 접속자의 개인 장치와 브라우저에 저장되는 텍스트 파일(정보 패킷)
웹사이트는 쿠키를 통해 접속자의 장치를 인식하고 접속자의 설정 / 과거 이용내역에 대한 일부 데이터 저장

## 세션
클라이언트와 웹서버간의 통신 연결에서 두 개체의 활성화된 접속
클라이언트가 웹 서버에 요청해 처음 접속하면 JSP나 ASP 엔진은 요청한 클라이언트에 대하여 유일한 ID를 부여하는데, 이 ID가 세션이다

## 세션 쿠키

## REST API 
REST 구조를 갖는 API

### (참고) API란? 어플리케이션 소프트웨어를 구축하고 통합하기 위한 정의 / 프로토콜 세트 (Application Programming Interface)

REST의 요소 - Method, Resource, Representation of Resource
Method 
- POST : POST를 통해 해당 URI를 요청하면 리소스를 생성한다.
- GET : GET을 통해 해당 리소스를 조회하고 해당 도큐먼트에 대한 자세한 정보를 가져온다.
- PUT : PUT을 통해 해당 리소스를 수정한다.
- DELETE : DELETE를 통해 리소스를 삭제한다.

* POST는 한번 수행했을 때와 여러번 수행했을 때 결과가 같지 않을 수 있다.


## GET과 POST의 비교 

"GET" - 데이터 요청할 때 그 데이터가 HTTP Request Message의 헤더 부분의 url에 담아서 전송됨
->요청 시 url상의 ? 뒤에 데이터가 붙어 request가 보내짐. 즉 보낼 수 있는 데이터 크기가 제한적이고 보안 측면에서 적절하지 않을 수 있음.
예시) https://www.twitter.com/search?q=제니%20gif

"POST" - Body 부분에 데이터를 담아 전송함. 즉 서버의 값이나 상태 등을 변경하기 위해 사용됨
예시) 본인이 게시글을 쓰는 경우


## Lord of SQL Injection

### 1번 gremlin 풀이
php언어의 preg_match는 preg_match('/대조할 문자/','입력한문자',$matches);로 구현되는데, 대조할 문자 부분에 필터링할 문자열을 넣는다. 단 특수문자를 필터링할 때는 특수문자 앞에 |를 붙인다.
문제에서는 id와 pw에서 _과 .을 필터링하고 있는 것으로 보인다. 따라서 '나 ''를 필터링하고 있지 않고, if($result['id'])인 것으로 보아 id값에 아무 값이 들어와도 해결될 것으로 보인다.
따라서 ?id=lol 으로 id에 lol을 넣어줄 수 있다. (위 REST API의 "GET") 그리고 pw 부분을 해결하기 위해 1=1로 참을 만들어주고 그 뒤를 주석처리 해주면 되는데, sql에서는 주석처리 #를 %23으로 한다고 한다.
따라서 url 뒷부분에 ?id=lol'or 1=1 %23을 넣어주면 된다. 그럼 1=1은 참이므로 ok, 뒷부분(pw)은 주석처리되므로 해결된다.

-> 어떻게 막을 것인가? - ' 필터링만 해도 이 정도 수준의 해킹은 방지할 수 있을 것 같다.


### 2번 cobolt 풀이
1번과 마찬가지로 '나 ''를 필터링하고 있지 않다. id에 'admin'이 들어가면 해결되는 것으로 보이지만 pw 앞에 md5(찾아보니 해시 암호화라고 한다)라는 것이 있다.
하지만 id에 admin을 넣고 뒷부분을 모두 주석처리하면 상관없을 것으로 보인다. 따라서 ?id=admin'%23을 넣어주면 해결된다.
(비교적 쉬운 느낌?...)

-> 어떻게 막을 것인가? 마찬가지로 ' 필터링을...


### 3번 goblin 풀이
이번에는 '와 ''를 모두 금지하고 있다는 것과 id에 admin값이 들어오면 solve되는 것, 문제의 "no=" 이 세 부분이 가장 먼저 눈에 들어왔다.
그리고  no={$_GET[no]}부분을 봤을 때 no 값은 입력을 받는 것 같았다 (get이라서...) 수업에서 사용자가 입력하는 것들이 가장 문제가 된다고 했던 것이 떠올라서 이 부분을 이용해야겠다는 생각이 들었다.
그래서 no에 0을 넣어보니 놀랍게도 아무 일도 일어나지 않았다. (no=2,3,...일 때도) 그래서 1을 넣었더니 "Hello guest"라는 문구가 떴다. guest는 no번호가 1으로 지정되어 있는 것 같다.
id='admin'을 하려고 보니 '가 필터링되어 있어서 수업에서 들었던 대로 guest를 16진수로 변환해 ('를 사용하지 않기 위해) 0x61646d696e로 넣어 보았다. 
일단 guest가 아니고 admin이 되어야 하니까 no 값은 1이 아니어야 해서 그냥 0을 넣었다. ?no=0 or id=0x61646d696e를 넣었더니 Hello admin이 뜨면서 클리어가 되었다~

-> 어떻게 막을 것인가? '을 변환한 여러 값들도 막아 둬야 할 것 같고, id에 admin값이 들어오면 무조건 solve되는 점도 해결해야 할 것 같다.
