Overview

대학교 간식사업 간 발생하는 애로사항을 해결하기 위해 쿠폰 발급 서비스 DASH를 개발했습니다.

Responsibilities

  • · UX/UI 디자인
  • · 프론트엔드 개발

Tech Stack

React TypeScript Figma

Problem & Goals

Problem Context

- 발행 요청자 (소학회, 총학생회 등): 쿠폰 양식 자체제작 필요, 업장에 개별적으로 연락 후 양해 필요, 쿠폰 수요 미충족 시 예산손실 등의 문제
- 발행자 (소상공인): 쿠폰 위조 우려, 발행 후 근무자 간 사용 집계 및 인수인계 어려움

Solution Breakdown

Step 1

Discovery

지류쿠폰 형식은 유지하며 온라인 배부 및 쿠폰 유효성 검증 등의 기능을 플랫폼에서 관리할 수 있도록 구성하는 아이디어를 고안헸습니다.
‘등록번호’와 ‘결제번호’를 분리하여 시스템 내에서 유효성이 확인된 쿠폰에 대해서만 결제번호를 발급/결제 활성화할 수 있게 했습니다.
실제 학생회, 소학회 임원과 인터뷰하고 기존 지류 쿠폰 플로우를 여정지도로 재구성했습니다.
쿠폰 위조, 승인 지연, 정산 오류를 핵심 페인포인트로 정리하고 이를 해결할 기능 리스트를 도출했습니다.

Step 2

Architecture

[Front-end]
- React 웹앱을 통해 관리자 및 사용자에게 실시간 인증 및 쿠폰 관리/통계 기능을 제공합니다.
- HTML5-QRCode를 사용해 모바일 웹에서 빠르고 안정적인 실시간 QR 스캔 기능을 구현합니다.

[Back-end]
- Spring & Java 기반의 Rest API를 개발해 쿠폰 발행 및 결제 활성화를 담당합니다.
- AWS 환경에 Docker & k8s를 활용해 컨테이너 기반의 자동화 배포 및 분산 처리 환경을 구축했습니다.
- Master 노드에는 Argo를 활용한 CI/CD 파이프라인과 워크로드 관리를 설정하고, Worker 노드에서는 실제 API 및 서비스가 구동되도록 합니다.
- MySQL을 사용해 핵심 데이터를 저장/관리합니다.

[Blockchain Module]
- Solidity를 통해 쿠폰 유효성 검증 및 소유권 관리를 위한 스마트 컨트랙트를 Ethereum 네트워크에 배포합니다.

Step 3

Implementation

[Front-end]
- Tailwind CSS를 활용하여 직관적이고 반응형(Responsive)인 UI 컴포넌트들을 구축하여, 관리자 및 사용자 경험(UX)을 최적화했습니다.
- Zustand를 사용하여 전역 상태 관리를 구현함으로써 대시보드의 복잡한 데이터 흐름을 효율적으로 관리했습니다.
- React Query를 도입하여 API 캐싱 전략을 수립하고, 서버 상태를 효과적으로 관리하여 불필요한 API 호출을 줄이고 사용자 체감 속도를 향상시켰습니다.

[Back-end & Infra]
- Spring Boot를 사용하여 RESTful API를 개발하고, 명세에 따른 일관적인 JSON 데이터 형식 처리를 구현했습니다.
- Kubernetes 환경에 배포 매니페스트를 작성하고, Argo CD를 통해 GitOps 기반의 자동 배포 파이프라인을 구축하여 서비스의 민첩성(Agility)과 안정성(Reliability)을 확보했습니다.
- Docker Compose를 활용하여 개발 환경과 테스트 환경을 로컬에서 쉽게 구축 및 통합할 수 있도록 지원했습니다.

[BlockChain]
- Solidity 스마트 컨트랙트를 작성하여 쿠폰의 발행 및 소각 로직, 유효성 검증 로직을 구현했습니다.
- 유효성 검증 및 서명 절차를 스마트 컨트랙트 로직에 추가하여, 쿠폰 결제 과정에서의 **보안성(위변조 방지)**을 극대화했습니다.

Step 4

Launch & Measure

베타 릴리스 후 SOFTCON 2024 캡스톤디자인 경진대회 당일, 학교 앞 소상공인과 실제로 협업해 쿠폰을 발급하고, 실제로 서비스를 이용해 상품을 교환할 수 있도록 했습니다.
